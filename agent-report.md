# Agent: Report Agent

## Role

The Report Agent is the final stage of the pipeline. It reads all generated test files from the Memory Store, uses Claude to score their quality and assess the PR risk level, then produces a structured Markdown report and posts it as a comment on the GitHub Pull Request.

---

## Responsibilities

1. Read all generated test bundles from the Memory Store.
2. Call Claude once to score test quality and produce a risk assessment.
3. Determine a Go/No-Go recommendation based on scores and risk signals.
4. Generate a Markdown report summarising the results.
5. Post the report as a comment on the GitHub PR via Octokit.
6. Save the full JSON report to `./output/report-pr-<number>.json`.
7. Save the Audit Log to `./output/audit_log.json`.
8. Exit with code `0` (go / go-with-caution) or `1` (no-go) for CI integration.

---

## Input (read from Memory Store)

```typescript
interface ReportAgentInput {
  pr: PullRequestMeta;
  changedFiles: ChangedFile[];
  generatedFiles: GeneratedTestBundle[];
}
```

---

## Output

```typescript
interface ReportAgentOutput {
  overallRisk: 'low' | 'medium' | 'high' | 'critical';
  riskReasons: string[];
  testScores: TestScore[];
  goNoGo: 'go' | 'go-with-caution' | 'no-go';
  prCommentUrl: string;
  reportPath: string;
}

interface TestScore {
  filePath: string;
  type: 'unit' | 'ui' | 'perf';
  score: number;       // 0–10 average across correctness, coverage, quality
  notes: string;
}
```

---

## Claude AI Usage

The Report Agent makes **one Claude call** — it sends all generated test content together and asks for a combined quality score and risk assessment.

### Prompt Pattern

```typescript
const systemPrompt = `
You are a senior QA architect reviewing AI-generated tests.
Score each test file and assess the overall PR risk.
Output ONLY valid JSON — no markdown, no code fences.
`

const userPrompt = `
PR: #${pr.number} — "${pr.title}"

Changed files:
${changedFiles.map(f => `- ${f.path} (${f.category.join(', ')}, ${f.linesAdded} lines added)`).join('\n')}

Generated tests:
${generatedFiles.map(b => `
=== ${b.sourcePath} ===
VITEST: ${b.vitest?.content ?? 'not generated'}
PLAYWRIGHT: ${b.playwright?.content ?? 'not generated'}
K6: ${b.k6?.content ?? 'not generated'}
`).join('\n')}

Respond with:
{
  "scores": [
    {
      "filePath": "output/unit/authService.test.ts",
      "type": "unit",
      "score": 8,
      "notes": "Good coverage of happy path and error cases, mocking correct"
    }
  ],
  "overallRisk": "medium",
  "riskReasons": ["authService.ts handles JWT signing", "200+ lines changed"],
  "goNoGo": "go-with-caution"
}
`
```

---

## Go/No-Go Logic

| Condition | Recommendation |
|-----------|---------------|
| Risk = low AND avg score ≥ 8 | `go` |
| Risk = medium OR avg score 6–7 | `go-with-caution` |
| Risk = high OR any score < 5 | `no-go` |
| Risk = critical | `no-go` |

---

## Risk Assessment Signals

Claude evaluates risk based on these signals in the changed files:

| Signal | Risk Impact |
|--------|------------|
| Auth / token / password / crypto in file path or content | High |
| Payment / billing / subscription logic | High |
| More than 200 lines changed in a single file | Medium |
| Core service modified (many potential dependents) | Medium |
| Config or environment variable changes | Medium |
| UI-only or utility changes | Low |

## PR Comment Format

```markdown
## 🤖 AI Test Automation Report

**PR #42** — `feat: add user authentication endpoints`
**Risk Level:** 🟡 Medium | **Go/No-Go:** 🟡 Go with caution
**Files analyzed:** 4 | **Test files generated:** 6

---

### Generated Tests

| Source File | Vitest | Playwright | k6 |
|------------|--------|------------|----|
| authService.ts | ✅ authService.test.ts (score: 8/10) | — | ✅ authService.k6.js |
| LoginForm.tsx | ✅ LoginForm.test.ts | ✅ login.spec.ts (score: 7/10) | — |

---

### Risk Assessment
- ⚠️ `authService.ts` — JWT signing logic, high security impact
- ✅ `LoginForm.tsx` — UI component, low risk

---

### Recommendations
- Review `authService.test.ts` manually — token expiry edge cases may need refinement
- Add integration test for the full login → refresh → logout flow

---

*Generated by Intelligent Test Automation Agent | [Full report](./output/report-pr-42.json)*
```

---

## Error Handling

- If GitHub comment post fails → save report locally and log the error; do not fail the pipeline.
- If Claude scoring call fails → use a default score of `5/10` with note *"scoring unavailable"* and continue.
- Always write the JSON report to disk regardless of GitHub API status.
